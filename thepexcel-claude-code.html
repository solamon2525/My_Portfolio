ในมุมมองของคนที่ไม่ใช่ Programmer

✍️ Sira Ekabut 📅 26 ธันวาคม 2568 🔗 ต้นฉบับ
TL;DR

เจ้าของเว็บ ThepExcel ที่ไม่ใช่ programmer ใช้ Claude Code ทุกวัน เพราะมันคือ AI Agent ที่อ่าน/เขียนไฟล์โดยตรง ทำงานได้หลากหลาย customize ได้สูง และมี ecosystem (CLAUDE.md, Skills, MCP, Hooks, Subagents) ที่ทรงพลังกว่า chatbot ทั่วไปมาก

💡

จุดเริ่มต้น

ผู้เขียนเป็นคนสอน Excel / Power BI เขียน Python งูๆ ปลาๆ ใช้ Obsidian จดโน้ต ปัญหาคือต้อง copy-paste โน้ตไปใส่ ChatGPT ทุกครั้ง พอเจอ Claude Code ก็แค่เปิด folder พิมพ์ "ช่วยสรุปโน้ตเรื่อง X" — มันไปอ่านไฟล์เองเลย

🤖

ไม่ใช่แค่เขียนโค้ด

มันคือ AI Agent ที่ทำงานได้หลากหลาย:

  • เขียน Content สำหรับเว็บ
  • จัดการ JSON files หลายร้อยไฟล์
  • Research ข้อมูลจากเว็บ
  • สร้าง Automation scripts
  • ทำงานกับ Obsidian notes, Excel, PDF, รูปภาพ ฯลฯ
หลักคิด: อะไร Code ได้ก็ให้ LLM Code, ที่เหลือให้ LLM Generate เอา — แต่สุดท้ายเราตรวจผลลัพธ์อยู่ดี
🔀

3 รูปแบบการใช้งาน AI

💬Chat Interface
ChatGPT, Claude.ai — ใช้ง่าย แต่ต้อง copy-paste ไปมา AI ไม่เห็น context งาน
🖥️IDE Integration
VS Code, Cursor, Windsurf — AI ฝังใน Code Editor เหมาะโปรแกรมเมอร์
⚡CLI — ตัวเปลี่ยนเกม ★
ทำงานระดับ OS เข้าถึงทุกอย่างในเครื่อง Claude Code เป็น first-party จาก Anthropic ได้ feature ใหม่ก่อนใคร
⚙️

ติดตั้ง & เริ่มต้น

รองรับ macOS, Linux, Windows (Native PowerShell / WSL) — ผู้เขียนแนะนำ WSL บน Windows เพราะ AI ถนัดคำสั่ง Linux

  1. สมัคร Claude Pro ที่ claude.ai
  2. ติดตั้ง Node.js จาก nodejs.org
  3. ติดตั้ง Claude Code ผ่าน terminal (คำสั่งด้านล่าง)
  4. Login ด้วย claude login
  5. เปิด folder ที่ต้องการแล้วรัน claude
⌨️

คำสั่งทั้งหมด (Command Reference)

1. ติดตั้ง Claude Code

npm install -g @anthropic-ai/claude-code

2. Login

claude login

3. เปิดใช้งาน (cd ไปที่ folder ก่อน)

claude

4. YOLO Mode — ทำงานโดยไม่ต้องกด approve ทุก action

claude --dangerously-skip-permissions
⚠️ ข้อควรระวัง: ใช้ได้ปลอดภัยเพราะ commit/push GitHub บ่อย ถ้า AI ทำพังก็ revert ได้ — เหมาะกับโปรเจกต์ที่ทำคนเดียว

5. เรียก Slash Command

# พิมพ์ / แล้วตามด้วยชื่อ command ที่สร้างไว้
/full 10 excel   # เขียน Excel function 10 ตัว แล้ว publish
/write 5 dax    # เขียน JSON อย่างเดียว ไม่ publish
/sync-related   # Sync related functions ขึ้น WordPress

6. บันทึกข้อมูลเข้า CLAUDE.md ด้วย # ใน prompt

# พิมพ์ # นำหน้าในช่อง prompt ของ Claude
#ผมชื่อศิระ
#ภาษาที่ใช้: ไทย ศัพท์เทคนิคเป็นอังกฤษ

7. Import ไฟล์อื่นใน CLAUDE.md

# ใน CLAUDE.md ใช้ @ นำหน้า path
@README.md
@docs/style-guide.md

8. ติดตั้ง Skill จาก GitHub (user-level)

git clone https://github.com/ThepExcel/agent-skills.git
mkdir -p ~/.claude/skills
cp -r agent-skills/deep-research ~/.claude/skills/
rm -rf agent-skills

9. ติดตั้ง Skill (project-level)

git clone https://github.com/ThepExcel/agent-skills.git
mkdir -p .claude/skills
cp -r agent-skills/deep-research .claude/skills/
rm -rf agent-skills

10. Plugin Marketplace

# Add marketplace (ทำครั้งเดียว)
/plugin marketplace add ThepExcel/thepexcel-claude-plugins

# Install skill ที่ต้องการ
/plugin install deep-research@thepexcel-claude-plugins
/plugin install triz@thepexcel-claude-plugins

# ดู skill ทั้งหมดใน marketplace
/plugin list thepexcel-claude-plugins

Features สำคัญ

📋

CLAUDE.md — กฎที่โหลดอัตโนมัติทุก session

ไฟล์ Markdown ที่ Claude อ่านทุกครั้งที่เปิด session อัตโนมัติ เหมือนใส่ความทรงจำให้ AI ล่วงหน้า

# ตัวอย่าง CLAUDE.md

# Core Principles
1. Never Guess - อ่านโค้ดก่อนตอบ อย่าเดา
2. Find Root Cause - หาสาเหตุที่แท้จริง
3. Minimize Changes - ทำเฉพาะที่ขอ ไม่ over-engineer

# Project Structure
- /functions/ = JSON สำหรับ Function Documentation
- /tools/ = Python scripts สำหรับ automation
- /theme/ = WordPress Block Theme
💡 Tips: เก็บ CLAUDE.md ให้กระชับ เพราะโหลดทุก session (กิน Token!) ส่วน docs ละเอียดให้ @import จากไฟล์แยก
⚡

Slash Commands — Shortcut Prompt ที่ใช้บ่อย

เหมือน Macro ใน Excel — กำหนด Prompt เป็นขั้นตอนใน .claude/commands/ แล้วเรียกด้วย /

# สร้างไฟล์ .claude/commands/full.md
mkdir -p .claude/commands
# แล้วเขียน Markdown คำสั่งใน full.md
📢 อัปเดต ม.ค. 2026 (v2.1.3): Skills และ Slash Commands ถูก merged กัน ใช้ได้ข้าม platform (Claude.ai, API, Gemini CLI, Codex) — แนะนำให้สร้าง Skills เสมอ
🧠

Skills — SOP ที่ Claude หยิบมาใช้เองอัตโนมัติ

ต่างจาก Slash Command ที่เราต้องพิมพ์เรียก — Skills คือ Claude เลือกใช้เองเมื่อเจอสถานการณ์ที่เหมาะสม

# ตัวอย่าง Skill definition
skill: function-explainer
description: "สร้าง function documentation สำหรับ Excel/Power Query"
trigger: เมื่อพูดถึงการเขียน documentation
👥

Subagents — AI ย่อยที่ทำงาน parallel

Claude หลักคือ Project Manager — spawn Subagents ย่อยทำงานพร้อมกัน แต่ละตัวมี context แยกกัน

# เมื่อสั่ง /full 10 excel
→ Spawn function-writer #1 → SUMIF  ┐
→ Spawn function-writer #2 → VLOOKUP ├ พร้อมกัน!
→ Spawn function-writer #3 → INDEX  ┘
# 30 นาที → 5-8 นาที
🎯 เลือกใช้ model ตามงาน: งานง่าย → Haiku (เร็ว ถูก), งานซับซ้อน → Sonnet (ฉลาดกว่า)
🔌

MCP — เชื่อมต่อระบบภายนอก

Model Context Protocol คือ "ปลั๊กไฟสากล" ให้ Claude เชื่อมต่อระบบภายนอกได้

MCP Serverทำอะไรได้
Filesystemอ่าน/เขียนไฟล์นอก project folder
GitHubจัดการ repo, PR, issues
Slackอ่าน/ส่งข้อความใน Slack
Notionอ่าน/เขียน Notion pages
Power BIจัดการ reports, datasets, DAX
⚠️ MCP ให้ AI เข้าถึงระบบจริงได้ — ระวังโดยเฉพาะ MCP ที่เขียน/ลบข้อมูลได้
🎣

Hooks — Automation เมื่อเกิด Event

ตั้งค่าใน ~/.claude/settings.json (user) หรือ .claude/settings.json (project)

Eventเมื่อไหร่
SessionStartเมื่อเริ่ม session ใหม่
SessionEndเมื่อ session จบ (v2.1+)
PreToolUseก่อน Claude จะใช้ tool
PostToolUseหลัง Claude ใช้ tool เสร็จ
UserPromptSubmitเมื่อ user ส่ง prompt
PermissionRequestเมื่อขอ permission (v2.1+)
SubagentStopเมื่อ subagent ทำงานเสร็จ (v2.1+)
PreCompactก่อน compact context (v2.1+)
Stopเมื่อ Claude หยุดทำงาน

Hook ตัวอย่าง — ป้องกันแก้ไฟล์ .env:

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit",
      "command": "if echo '$FILE_PATH' | grep -q '.env'; then echo 'BLOCKED!'; exit 1; fi"
    }]
  }
}

Hook ตัวอย่าง — แจ้งเตือนเมื่องานเสร็จ:

{
  "hooks": {
    "Stop": [{
      "command": "notify-send 'Claude Code' 'งานเสร็จแล้ว!'"
    }]
  }
}
📦

Plugins — รวมทุกอย่างเป็น Package

รวม CLAUDE.md + Skills + MCP + Hooks เป็น package เดียว แชร์ให้คนอื่นได้

my-plugin/
├── CLAUDE.md           # กฎทั่วไป
├── .claude/
│   ├── skills/       # Skills
│   ├── commands/     # Slash commands
│   └── settings.json # MCP + Hooks
└── README.md

Workflow จริง: Function Explainer

พิมพ์ /full 10 excel แล้ว Claude ทำทั้งหมดนี้อัตโนมัติ:

1. Main Agent → หา function ที่ต้องปรับปรุงใน database
2. Main Agent → เรียก Function Explainer Skill
3. Skill → spawn SubAgents 10 ตัวพร้อมกัน
4. SubAgents → Research + เขียน Documentation แต่ละตัว
5. Main Agent → Validate JSON format (แก้เองถ้าไม่ผ่าน)
6. Main Agent → Publish ขึ้น WordPress ผ่าน API

# ผลลัพธ์: 10 functions เสร็จใน 3-5 นาที (ทำเองครึ่งวัน)
